## 1. Definição do MVP
* **Dentro do MVP:** Cadastro dos medicamentos, registro de vendas, baixa automática de estoque, consulta de preços e cadastro simples de clientes para desconto.
* **Fora do MVP:** Gestão de pedidos com fornecedores, sistema de delivery, relatórios financeiros complexos e programas de pontos avançados.
* **Justificativa:** A escolha foca em garantir a integridade do inventário e a agilidade no atendimento ao cliente, que são as funcionalidades críticas para o funcionamento inicial do negócio.

---
## 2. Regras de Negócio (RN)
* **RN01:** Medicamentos controlados exigem registro de receita.
* **RN02:** Bloqueio de venda se o estoque estiver vazio.
* **RN03:** Desconto de 5% automático para clientes cadastrados.
* **RN04:** Todos os medicamentos devem ter data de validade.
* **RN05:** Vendas canceladas estornam produtos ao estoque.


## 3. Requisitos Funcionais (RF)
* **RF01 — Controle de Acesso:** O sistema deve permitir o login e autenticação de funcionários quando colocar usuário e senha.
* **RF02 — Gestão de Medicamentos:** O sistema deve permitir o cadastro de novos produtos, incluindo preço, lote e validade.
* **RF03 — Consulta de Itens:** O sistema deve oferecer busca por nome ou código para consultar preços e saldo em estoque.
* **RF04 — Registro de Vendas:** O sistema deve permitir a inclusão de múltiplos itens em uma única transação de venda.
* **RF05 — Cálculo de Transação:** O sistema deve calcular o subtotal, aplicar descontos e informar o valor do troco.
* **RF06 — Sincronização de Estoque:** O sistema deve realizar a baixa automática das quantidades no estoque após a confirmação da venda.
* **RF07 — Gestão de Clientes:** O sistema deve permitir o cadastro de clientes utilizando o CPF como identificador único.
* **RF08 — Geração de Comprovante:** O sistema deve gerar um comprovante de venda detalhando itens e valores pagos.

## 4. Requisitos Não Funcionais (RNF)
* **RNF01 — Performance:** O tempo de resposta para busca de produtos deve ser inferior a 1 segundo.
* **RNF02 — Usabilidade:** A interface deve ser responsiva para funcionamento em desktops e tablets.
* **RNF03 — Segurança:** Todas as senhas de acesso devem ser armazenadas utilizando criptografia.
* **RNF04 — Confiabilidade:** O sistema deve realizar backups automáticos da base de dados a cada 24 horas.
